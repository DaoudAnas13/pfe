\chapter{Project Planning}

\section{Introduction}

This chapter presents the project planning strategy adopted to deliver the RM platform in a controlled, iterative, and traceable manner. It explains the SCRUM adaptation used in a single-developer context, the software and engineering tools selected for implementation, and the architecture decisions that structure the system at physical and logical levels.

The planning objective was to guarantee progressive value delivery while preserving software quality attributes, especially security, maintainability, and performance.

\section{SCRUM Methodology}

\subsection{Method Selection Rationale}

SCRUM was selected because the project combined evolving business needs, multi-platform implementation (web and mobile), and high uncertainty around field constraints. An iterative model provided three major advantages:

\begin{itemize}
\item frequent validation of deliverables with domain expectations,
\item early risk detection through short execution cycles,
\item and continuous reprioritization based on implementation feedback.
\end{itemize}

\subsection{SCRUM Roles in a Single-Developer Context}

Although the project was executed by one developer, SCRUM responsibilities were explicitly maintained:

\begin{table}[H]
\centering
\caption{SCRUM Roles and Practical Adaptation}
\label{tab:scrum_roles_adaptation}
\begin{tabularx}{\textwidth}{|l|X|X|}
\hline
\textbf{Role} & \textbf{Theoretical Responsibility} & \textbf{Practical Adaptation in this Project} \\
\hline
Product Owner & Define product vision and prioritize backlog & Backlog priorities were derived from the PRD and validated against business constraints \\
\hline
Scrum Master & Ensure process discipline and remove blockers & Sprint rituals, time-boxing, and risk monitoring were self-managed \\
\hline

\end{tabularx}
\end{table}

\subsection{Sprint Cadence and Governance}

Each sprint followed a three-week cycle:

\begin{enumerate}
\item Sprint planning and backlog selection,
\item architecture and design detailing,
\item backend and frontend implementation,
\item integration and test execution,
\item sprint review and retrospective notes.
\end{enumerate}

This cadence was repeated for five sprints, covering foundation, core workflow, evidence handling, approval governance, and reporting optimization.

\subsection{Development Environment}

The project environment was organized to support repeatable development and testing:

\begin{itemize}
\item IDEs: IntelliJ IDEA for backend, VS Code for frontend/mobile work,
\item version control: Git-based workflow with incremental commits per sprint objective,
\item local execution: service startup profiles for backend APIs, Angular UI, and Flutter client,
\item database environment: PostgreSQL instance with versioned schema evolution.
\end{itemize}

\subsection{Supporting Software}

\begin{itemize}
\item Postman for API verification and endpoint scenario testing,
\item pgAdmin for schema and data inspection,
\item Swagger/OpenAPI for endpoint discoverability,
\item Docker tooling for packaging and deployment preparation,
\item spreadsheet tools for validating report outputs generated by Apache POI.
\end{itemize}

\subsection{Frontend Architecture (Angular)}

The Angular application adopts a modular architecture:

\begin{itemize}
\item \textbf{Core Module}: authentication guard, interceptors, and shared services.
\item \textbf{Feature Modules}: intervention dashboard, template management, reporting, administration.
\item \textbf{Shared Module}: reusable components, pipes, validators, and UI elements.
\item \textbf{Routing Layer}: role-aware navigation and guarded access.
\end{itemize}

\subsection{Inter-service Communication}

Communication is HTTP REST based and organized by endpoint families. Key interaction patterns include:

\begin{enumerate}
\item token-based authenticated requests from web/mobile clients,
\item domain service orchestration for lifecycle transitions,
\item asynchronous notification dispatch after state changes,
\item generated report storage and retrieval workflows.
\end{enumerate}
